Imagens dinâmicas

Tente abrir esse site em vários dispositivios diferentes ou simplesmente aumente e diminua o tamanho do seu navegador

Para fazer essas imagens dinâmicas usando a HTML5 é preciso três imagens de três tamanhos diferentes. No caso acima foram usados:

  1. 300x300px
  2. 700x700px
  3. 1000x1000px
Dessa forma, por meio do código HTML o browser vai exibir a imagem com o width (largura) adequada ao tamanho da tela do dispositivo.
O código segue a seguinte expressão usando a tag <img> aninhada a tag <picture>: 

<picture>
    <source media="(max-width: 750px) src="imagens/foto-p.png" type="image/png">
    <source media="(max-width: 1050px) src="imagens/foto-m.png" type="image/png">
    <img src="imagens/foto-g.png" alt="imagem flexível">
</picture>
Como se pode a imagem base trabalhada é a de 1000x1000px. Portanto, se fosse alterado a ordem das tags source a imagem de 300x300px não iria aparecer.Dessa forma a ordem a ser seguida, nesse código é:
  1. source imagem menor
  2. source imagem média
  3. imagem base

Por sua vez, também é possível fazer da maneira inversa ao trabalhar-se atribuido ao source media o min-width conforme visto acima: 


<picture>
    <source media="(min-width: 100px) src="imagens/foto-p.png" type="image/png">
    <source media="(min-width: 700px) src="imagens/foto-m.png" type="image/png">
    <img src="imagens/foto-p.png" alt="imagem flexível">
</picture>
Desta forma, a imagem base é a imagem de 300x300px e, se fosse alerada a ordem das tags source a imagem de 1000x1000px não iria aparecer. Dessa forma a ordem a ser seguida, é:
  1. source imagem maior
  2. source imagem média
  3. imagem base

Reproduzir áudios

Há duas formas de reproduzir um áudio usando HTML5 a primeira é:

  1. Usando a tag <audio>:
    2. 
<audio src="(url da media)" controls autoplay></audio>

    Perceba que nem todo browser é compatível com essa tag ou reproduz todos os formatos de áudio. Os formatos mais aceitos são .mp3, .wav e .ogg

  2. Outra forma é:
    3. 
<audio preload="metadata" autoplay controls>
    <source src="(url)" type="audio/mpeg">
</audio>

    Essa segunda forma de acrescer áudio ao site usando HTML5 torna o áudio compatível com uma maior variedade de browsers. Contudo, nem todos os browsers aceitam o formato .mp3, por essa razão pode-se colocar abaixo da tag source os outros dois tipos de áudios suportados (.wav e .ogg). Ademais, também é possível colocar uma mensagem para o caso do browser não suportar nenhuma das opções.

    O preload na tag audio pode ser atribuido com "metadata", "auto" e "none".

    • metadata: carrega apenas as informações
    • auto: sem que o cliente clique o áudio já está carregado
    • none: somente quando o cliente clicar é que será carregado as informações e o arquivo
    Também é possível atribuir o parâmetro loop a tag audio. O loop faz o audio se repetir infinitamente uma vez tocado.
